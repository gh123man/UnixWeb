Category: File Management
Definition: changes a group ownership of a file or of many files.

Options:

  • -C, --changes
    • reports when a change is made
  • -f, --silent, --quiet
    • makes it so error messages are not shown
  • -v, --verbose
    • prints out details/diagnostic for every file processed/changed
  • -h, --no-dereference
    • affects symbolic links instead of any referenced file
    • can only use this if you can change symlink ownership
  • --no-preserve-root
    • do not treat '/' in a special way
    • default setting
  • --reference=FILENAME
    • use FILENAME's group settings rather than specifiying a group value
  • -R, --recursive
    • operates recursively on files and directories
    • these options modify how hierachy is affected when -R is specified
    • -H
      • if a command line arguement us a symbolic link, traverse it
    • -L
      • traverse every symbolic link to a directory
    • -P
      • do not traverse any symbolic links
      • this is the default
    • --help
      • display a help message
      • exits after message is displayed
    • --version
      • displays the output information

Uses/Examples:

  • chgrp users file.txt
    • changes the owning group of file.txt to the group named "users"
  • chgrp -hR admin /database/
    • changes the owning group of /database/ and all of its subdirectories & files to belong to admin
