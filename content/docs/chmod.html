Category: File Permissions
Definition: used to control access/permissions to files and directories. There are set "rules" in every hierachy that define who can access what file. These rules are known as file permissions that looks to read (r), write (w), and execute (x) the file. The abilities are set to define the permissions for three different groups: the user (aka the owner), the group (list of users belonging to the ownership's group), and the public (Other).

How to use permissions:
There are two ways that you can set permissions, by a combination of letter/operators or by a set of three numbers. The first way is by letters/operators: To define the different groups, you must a user class symbol:

  • u : user/owner
  • g : group
  • o : other/public
  • a : all aka (ugo)
To determine how to set, take away, or add symbols, you must use a permission operator:
  • = : allows the specificed permissions to be enabled for defined user class
  • + : adds/allows the specificed permissions to be enabled for defined user class
  • - : takes away/prohibits the specificed permissions to be enabled for defined user class
And lastly, you must determine the permissions themselves.
  • r: the ablity to read the contents of the file
  • w: the ability to change the contents of the file (write)
  • x : the ability to run the contents of the file as a program
Examples of class/operator/permissions combinations:
  • u=rwx
    • user/owner can read, write, and execute the file
  • o+r
    • other/public can other read the file as well
  • u-w
    • user/owner can no longer change the contents of the file (write)
  • u-w
    • user/owner can no longer change the contents of the file (write)


The second way is by three numbers. Each number represents a permission.
  • 4: read
  • 2: write
  • 1 : execute
  • 0 : no permission
To set permissions for each user class, we use a three digit number to represent them all. for example 654. 6 refers to the owner/user class, 5 to the group class, and 4 to the other/public class. How did we get 6 & 5? Simple really, to represent multiple permissions to each other.
So for the user class we have 6 which means we combined the read(4) and the execute(2) permissions together (4+2 = 6).
For the group, it was the read(4) and execute(1) permissions to get 5.
The other/public class, it's easy, we simply have 4 for read.
Common combinations are: 644 for files and 755 for directories.

Options:

  • -C, --changes
    • reports when a change is made
  • -f, --silent, --quiet
    • makes it so error messages are not shown
  • -v, --verbose
    • prints out details/diagnostic for every file processed/changed
  • -h, --no-dereference
    • affects symbolic links instead of any referenced file
    • can only use this if you can change symlink ownership
  • --no-preserve-root
    • do not treat '/' in a special way
    • default setting
  • --reference=FILENAME
    • use FILENAME's group settings rather than specifiying a group value
  • -R, --recursive
    • operates recursively on files and directories
  • --help
    • display a help message
    • exits after message is displayed
  • --version
    • displays the output information

Uses/Examples:

  • chmod 644 file.txt
    • changes the permissions to read & write for the user, read only for the group & other users for file.txt
  • chmod u=rx file.txt
    • changes the permissions for the user/owner only to read and write the file.txt
  • chmod a=rw file.txt
    • changes the permissions for all user classes to be able to read and write the file.txt

Also note that when you type "ls - l" in your command line to see all directories/files, permissions will be displayed first.
