Category: File Management
Definition: to remove files or directories. In order to remove a directory, the -r option must be specified. Otherwise the rm command will only remove files.

Options:

  • -f, --force
    • ignores files that are nonexistant
    • does not prompt before removing
  • -i
    • gives a prompt before each removal
  • -I
    • prompts only once before removing if there are 3+ files being removed
    • prompts only once if removing reclusively
  • --interactive[=WHEN]
    • prompt removes as specified by WHEN
    • WHEN can be: always, once, never
    • default for WHEN is always
  • --one-file-system
    • for when you are removing a hierarchy recursively
    • says skip directories that is on a different file system than on the cmd line
  • --no-preserve-root
    • / is not treated in a special way
    • / is the root directory
  • --preserve-root
    • will not remove the root directory aka "/"
    • default setting
  • -r, -R, --recursive
    • removes directories and their contents recursively
  • -v, --verbose
    • explains what is happening for each action
  • --help
    • display help message and exits
  • --version
    • displays version and exits

Examples:

  • rm file1.txt
    • removes the file1.txt after you respond yes to the prompt
  • rm -f *
    • removes all the files in the current directory without a prompt
  • rm -r Sites
    • removes the Sites directory and all files in it
    • A prompt will appear for any write-protected files
  • rm -fr Sites
    • same as above, but no prompt will appear
  • rm -I *.txt
    • removes all text files
    • it shows a prompt IF there's more than three files
